# Secondary Market and Transfer Controls

## When to use this file

Use this file when the user asks about transfer restrictions, investor eligibility on secondary markets, compliance controls, or exit mechanics.

## Design considerations

- Decide if transfers are fully permissionless, gated, or restricted by status
- Define what happens on transfer failures
- Clarify how secondary-market activity is reported
- Document any lock-up, settlement, or redemption windows

## Safe defaults

- Treat transfer restrictions as part of a broader compliance process
- Make policy statuses observable and reviewable
- Avoid claiming that on-chain controls alone solve all exchange or securities concerns
